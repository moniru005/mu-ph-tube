Discuss the scope of var, let, and const

Var, let, and const are all different ways to declare variables in JavaScript.

Var: Var is the oldest keyword for declaring variables in JavaScript. It has a global scope, meaning that it can be accessed from anywhere within the script.

Let: Let was introduced with ES6 and allows you to declare variables with block-level scope, meaning they are only accessible within the block of code where they were declared.

Const: Const also was introduced with ES6 and allows you to declare variables whose values cannot be changed once assigned. They have block-level scope just like let declarations do.

Tell us the use cases of null and undefined

Use cases of null and undefined in JavaScript.

Null: Null is often used to represent an empty value or that a variable has not been initialized yet. For example, it may be used as the default value for a function parameter if no argument is provided.

Undefined: Undefined can mean that a variable has been declared but not assigned any value yet or that an object does not have any properties defined on it. It can also indicate the absence of a return statement in a function or the lack of an element in an array when trying to access it with its index number.
